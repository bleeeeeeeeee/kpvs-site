### Базовый рабочий цикл

- **Инициализация репозитория:**
  
  `git init`
  
  Создаёт локальный репозиторий в текущей папке.

- **Клонирование существующего репо:**
  
  `git clone <url>`
  
  Скачивает репозиторий и настраивает удалённый origin.

- **Проверка статуса:**
  
  `git status`
  
  Показывает изменения, индекс и текущую ветку.

- **Добавить файлы в индекс:**
  
  `git add .`
  
  Добавляет все изменённые/новые файлы в индекс.

- **Сделать коммит:**
  
  `git commit -m "Сообщение"`
  
  Фиксирует изменения из индекса с сообщением.

- **История коммитов:**
  
  `git log --oneline --graph --decorate`
  
  Краткая ветвящаяся история для наглядности.

### Ветвление и переключение

- **Создать новую ветку:**
  
  `git branch <branch>`
  
  Создаёт ветку без переключения.

- **Переключиться на ветку:**
  
  `git switch <branch>`
  
  Переходит на существующую ветку.

- **Создать и переключиться сразу:**
  
  `git switch -c <branch>`
  
  Создаёт ветку и сразу активирует её.

- **Список веток:**
  
  `git branch -a`
  
  Показывает локальные и удалённые ветки.

- **Слияние ветки:**
  
  `git merge <branch>`
  
  Вливает указанную ветку в текущую.

---

### Работа с удалённым репозиторием

- **Добавить удалённый:**
  
  `git remote add origin <url>`
  
  Привязывает удалённый репозиторий как origin.

- **Проверить удалённые:**
  
  `git remote -v`
  
  Показывает URLs для fetch/push.

- **Загрузить последние изменения:**
  
  `git pull`
  
  Получает и сливает изменения из удалённой ветки.

- **Отправить коммиты:**
  
  `git push`
  
  Отправляет текущую ветку на удалённый.

- **Первый push новой ветки:**
  
  `git push -u origin <branch>`
  
  Пушит ветку и настраивает tracking.

---

### Осмотр и сравнение изменений

- **Разница рабочих файлов:**
  
  `git diff`
  
  Показывает непроиндексированные изменения.

- **Разница индекса и последнего коммита:**
  
  `git diff --cached`
  
  Изменения, добавленные в индекс.

- **Показать изменения конкретного файла:**
  
  `git diff <path>`
  
  Дифф по одному файлу.

---

### Откат и восстановление

- **Отменить изменения файла до последнего коммита:**
  
  `git restore <path>`
  
  Возвращает файл из HEAD, не трогая индекс.

- **Убрать из индекса (оставить в рабочей папке):**
  
  `git restore --staged <path>`
  
  Снимает файл с индекса.

- **Запрятать незаконченные изменения:**
  
  `git stash -u`
  
  Сохраняет и очищает рабочую директорию, включая незакоммиченные файлы.

- **Вернуть спрятанное:**
  
  `git stash pop`
  
  Применяет и удаляет последний stash.

- **Жёсткий откат на коммит (осторожно):**
  
  `git reset --hard <commit>`
  
  Перемещает ветку и рабочие файлы к указанному коммиту.

### Полезно для VS Code

- **Открыть проект:**
  
  `code .`
  
  Запускает VS Code в текущей папке.

- **Встроенная панель Source Control:**

- **Формат коммитов:**
  
  Используй короткие, смысловые сообщения: тип изменения + область + кратко что сделано (например, feat: slider progress bar).

Если подскажешь, как у тебя настроен удалённый (GitHub/GitLab) и флоу веток, дам короткий чек-лист под твой процесс (feature → PR → review → merge).






**Основные состояния файлов:**
- **U (Untracked)** — новый файл, Git его ещё не отслеживает.  
- **M (Modified)** — файл изменён после последнего коммита.  
- **A (Added / Staged)** — файл добавлен в индекс (`git add`), готов к коммиту.  
- **D (Deleted)** — файл удалён.  
- **R (Renamed)** — файл переименован.  

---

**Разделы в панели Source Control:**
- **Changes** — изменения в рабочей папке, ещё не добавленные в индекс.  
- **Staged Changes** — изменения, которые уже добавлены в индекс и будут закоммичены.  

---

**Кнопки и действия:**
- **Stage Changes (+)** — добавить файл в индекс (`git add`).  
- **Unstage Changes (–)** — убрать файл из индекса (`git restore --staged`).  
- **Discard Changes** — отменить изменения в файле, вернуть к последнему коммиту.  
- **Commit (✓)** — создать коммит с выбранными изменениями.  
- **Push (↑)** — отправить коммиты на удалённый репозиторий.  
- **Pull (↓)** — получить свежие изменения из удалённого репозитория.  
- **Refresh (⟳)** — обновить статус файлов.  

---

**Ветки:**
- Внизу VS Code отображается текущая ветка.  
- Можно переключаться между ветками или создавать новые.  
- **Publish Branch** — отправить новую ветку на удалённый репозиторий.  

---

**История и сравнение:**
- При клике на файл VS Code показывает сравнение (diff): слева версия из последнего коммита, справа — текущие изменения.  
- Вкладка Timeline показывает историю изменений файла.
