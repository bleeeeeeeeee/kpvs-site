# Требования к адаптации проекта под PostgreSQL 18

## Текущая конфигурация

### База данных (из creation-bd.txt)
- **Версия:** PostgreSQL 18
- **Порт:** 5432
- **Схема:** 11 таблиц с полным набором индексов и ограничений
- **Особенности:**
  - Полнотекстовый поиск с поддержкой русского языка
  - Автоматическое обновление timestamp
  - CHECK-ограничения для slug и артикулов
  - Составные индексы для фильтрации

### Приложение (из db.js и server.js)
- **Текущая БД:** PostgreSQL (порт 5433)
- **ORM:** pg (не ORM, а raw SQL)
- **Пул подключений:** 10 соединений
- **Таймауты:** 30s idle, 2s connection
- **База данных:** kpvs_db

### 2. Сравнение схем БД

#### Текущая схема (из запросов в db.js):
- `products` - с полями: id, name, slug, description, price, image_path, gender_code, category_code, created_at
- `categories` - с полями: code, name, parent_code, level, sort_order, description, is_active, gender
- `genders` - отдельная таблица
- `tags` - с полями: code, name, icon, color, sort_order
- `product_tags` - связь products-tags
- `product_sizes` - связь products-sizes
- `product_materials` - связь products-materials
- `product_images` - с полями: product_id, image_path, is_main, sort_order
- `sizes` - отдельная таблица
- `materials` - отдельная таблица

#### Новая схема (из creation-bd.txt):
- `categories` - иерархическая структура с parent_id
- `brands` - новая таблица
- `size_types` - новая таблица
- `sizes` - привязка к size_types
- `colors` - новая таблица
- `products` - расширенная структура: art, materials, season, gender, is_active
- `product_attributes` - новая таблица для характеристик
- `tags` - упрощенная структура
- `product_tags` - связь many-to-many
- `product_variants` - новая таблица для вариантов товаров
- `product_images` - расширенная структура

### 3. Ключевые различия

#### Структура продуктов:
- **Текущая:** Один товар с одним изображением, цены, гендер/категория через коды
- **Новая:** Товар с множественными вариантами (размер+цвет), множественные изображения, характеристики

#### Категории:
- **Текущая:** Плоская структура с parent_code
- **Новая:** Иерархическая с parent_id

#### Атрибутика:
- **Текущая:** Отдельные таблицы для размеров, материалов, тегов
- **Новая:** Унифицированная система через product_attributes и product_variants

#### Поиск:
- **Текущая:** ILIKE в запросах
- **Новая:** Полнотекстовый поиск через to_tsvector('russian')
