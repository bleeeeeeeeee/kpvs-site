# KPVS site

Магазин спецодежды: Node.js (Express), PostgreSQL, статические HTML/JS/CSS.

## Запуск

1. PostgreSQL и переменные окружения (см. `.env.example` при наличии или раздел ниже).
2. `npm install`
3. `npm start` — сервер слушает `PORT` (по умолчанию 3000).

Обязательно в production: `NODE_ENV=production`, `SESSION_SECRET` (не короче 24 символов), строка подключения к БД (`DATABASE_URL` или `PGHOST`/`PGUSER`/`PGPASSWORD`/`PGDATABASE`).

## Архитектурные решения

### После входа через Google (OAuth)

Параметр `next` сохраняется в сессии и после callback передаётся в `login.html` как query `next`. Принимаются только **внутренние пути**: одна ведущая `/`, без `//` (защита от protocol-relative редиректа на внешний сайт), без `\`, без нулевых байтов, длина ограничена. Иначе подставляется `/welcome.html`. Реализация: `sanitizeOAuthNextPath` в `server.js`.

### URL изображений товаров

В API создания/обновления товара поле `images[].url` проверяется так же, как на клиенте: допустимы относительные пути, начинающиеся с одной `/` (не `//`), или абсолютные `http:`/`https:` без userinfo в URL. Исключены `javascript:`, `data:`, `vbscript:`. Это снижает XSS через подставленные `src` и странные схемы.

### Учётные записи по умолчанию

Сиды с логинами `admin`/`superadmin` и паролями из кода выполняются **только если `NODE_ENV` не `production`**. В production первого администратора нужно создать вручную (SQL или будущий сценарий инициализации с секретом из окружения).

### Отладочный маршрут NDJSON

`POST /api/__debug_ndjson` доступен **только при `NODE_ENV` не равном `production`**. В production маршрут не регистрируется.

### Пароль к PostgreSQL в разработке

Если не заданы `DATABASE_URL` и `PGPASSWORD`, в не-production может подставляться пароль по умолчанию только для локальной разработки. На production без `DATABASE_URL` пароль не подставляется из кода.
